# 快捷键优化方案

## 问题分析

当前双击+长按模式存在的问题：
1. 操作复杂：需要 轻按 → 长按 两步操作
2. 时间阈值难以控制：用户很难在 300ms 内释放第一次按键
3. 不符合直觉：大多数语音输入软件都是"按下开始、松开停止"

## 推荐方案

### 方案：左右 Option 都用一次按键模式

```json
{
  "hotkeys": {
    "voice_input": "left_alt",      // 左 Option - 语音输入
    "quick_translate": "right_alt"  // 右 Option - 翻译（也用一次按键）
  }
}
```

### 交互流程

**左 Option（语音输入）**：
```
按下 → 立即开始录音
松开 → 等待200ms尾音 → 停止录音并识别
```

**右 Option（翻译）**：
```
按下 → 立即开始录音
松开 → 等待200ms尾音 → 停止录音并翻译
```

### 优点

1. ✅ **简单直观**：符合用户习惯
2. ✅ **即时响应**：按下立即开始录音
3. ✅ **左右分离**：两个功能独立，不会混淆
4. ✅ **容易实现**：代码逻辑简单

### 防误触措施

1. **防抖机制**：50ms 防抖，避免重复触发
2. **状态机保护**：录音中忽略新的按键
3. **尾音收集**：200ms 延迟，收集完整的语音

## 实施步骤

1. 修改配置文件，设置左右 Option
2. 简化 hotkey_manager.py，移除双击模式逻辑
3. 保留一次按键模式 + 尾音收集
4. 测试验证

## 参考资料

业界同类产品：
- **讯飞输入法**：F6/F7/F8 单键触发
- **Dragon Dictate**：自定义快捷键（多为单键）
- **macOS 自带听写**：双击 Fn 键（但这是系统级实现）

结论：**一次按键模式是语音输入的最佳实践**
